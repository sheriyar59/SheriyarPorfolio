<text>
**Design Movement**: Technical Brutalism meets Swiss Modernism

**Core Principles**:
- Raw, unpolished authenticity with precise grid systems
- Monospaced typography as a primary design element
- High contrast black/white with strategic accent colors
- Functional minimalism where every element serves a purpose

**Color Philosophy**: 
Start with a stark monochrome foundation (deep charcoal #1a1a1a and crisp white #fafafa) to represent technical precision. Introduce electric cyan (#00d9ff) as a singular accent color for interactive elements and key highlights, symbolizing the "signal" in network infrastructure—the data flow that Sheriyar manages. This creates a visual metaphor: the infrastructure is invisible (monochrome) until data flows through it (cyan).

**Layout Paradigm**: 
Asymmetric modular grid inspired by terminal windows and network diagrams. Content blocks are arranged like CLI output panels—left-aligned text blocks with right-aligned metadata, creating natural reading rhythms. Section breaks use horizontal rules that resemble network topology lines. The hero section features a large-scale grid overlay that subtly references network packet routing.

**Signature Elements**:
- ASCII-style decorative elements and dividers (using box-drawing characters)
- Monospaced code-style presentation for technical skills
- Network topology-inspired connecting lines between sections

**Interaction Philosophy**: 
Interactions should feel like command-line responses—immediate, precise, no-nonsense. Hover states trigger instant color shifts (white → cyan) with no easing curves. Clicking elements produces sharp, mechanical feedback. Scrolling reveals content with hard cuts rather than smooth fades, mimicking terminal screen refreshes.

**Animation**: 
Minimal, purposeful animations. Text appears with a subtle typewriter effect on initial load. Section transitions use hard wipes (left-to-right slides) rather than fades. Hover effects are instantaneous color inversions. Loading states use simple block-fill animations, never spinners. All timing is fast (100-200ms) to maintain the snappy, technical feel.

**Typography System**:
- Primary: IBM Plex Mono (400, 500, 600) for all body text, headings, and UI elements
- Secondary: IBM Plex Sans (600, 700) for large display text only
- Hierarchy through size and weight only, no font switching
- Generous letter-spacing (0.02em) for improved monospace readability
- Line height 1.6 for body text, 1.2 for headings

**Design Movement**: Neo-Memphis meets Contemporary Gradient Maximalism

**Core Principles**:
- Bold geometric shapes with playful asymmetry
- Rich, saturated gradients as primary visual language
- Layered depth through overlapping elements and shadows
- Energetic, confident personality that breaks corporate conventions

**Color Philosophy**:
Build a vibrant, warm palette that conveys energy and approachability. Primary gradient flows from deep coral (#ff6b6b) through warm orange (#ffa500) to golden yellow (#ffd700), representing the "heat" of network traffic and the warmth of reliable infrastructure. Secondary gradients use deep purple (#6a0dad) to electric blue (#4169e1) for contrast sections. Background is off-white cream (#fef9f3) to soften the intensity and improve readability. This palette communicates technical expertise without corporate coldness.

**Layout Paradigm**:
Diagonal-dominant composition where major sections are cut at 8-12 degree angles, creating dynamic visual flow. Content sits within rounded rectangular containers that overlap and cast soft shadows. The hero section features a large diagonal split—left side text, right side abstract geometric illustration. Scrolling reveals sections that slide in from alternating diagonal directions, creating a sense of movement and energy.

**Signature Elements**:
- Thick, rounded geometric shapes as decorative backgrounds
- Gradient-filled SVG blobs that shift subtly on scroll
- Oversized, bold section numbers with gradient fills

**Interaction Philosophy**:
Interactions should feel playful and responsive, like a well-designed mobile app. Buttons have a slight "bounce" on hover, growing 5% with a spring animation. Cards lift and cast deeper shadows when hovered. Clicking produces a subtle "squish" effect (scale down to 0.95, then back to 1). The overall feeling is tactile and satisfying, encouraging exploration.

**Animation**:
Generous, smooth animations with spring physics. Elements fade in with upward motion (20px translate) and slight rotation (2-3 degrees). Gradient backgrounds shift slowly on scroll using CSS background-position animation. Section reveals use staggered entrance animations—heading first, then content blocks with 100ms delays. Hover states animate over 300ms with cubic-bezier easing. Scroll-triggered animations activate when elements reach 20% viewport height.

**Typography System**:
- Display: Outfit (700, 800) for large headings and hero text
- Body: Inter (400, 500, 600) for all content and UI
- Accent: Space Grotesk (600) for section labels and metadata
- Hierarchy through dramatic size contrast (72px hero → 16px body)
- Tight line height (1.1) for display text, comfortable 1.7 for body
- Slight negative letter-spacing (-0.01em) on large display text for tighter feel

**Design Movement**: Japanese Minimalism with Editorial Magazine Influence

**Core Principles**:
- Abundant whitespace as the primary design element
- Vertical rhythm and precise alignment
- Subtle, sophisticated color restraint
- Content-first hierarchy with editorial pacing

**Color Philosophy**:
Adopt a serene, understated palette rooted in natural tones. Primary color is deep forest green (#2d4a3e) representing growth, stability, and environmental consciousness—qualities that align with infrastructure work. Accent with warm terracotta (#c77b58) for interactive elements and key highlights, adding warmth without aggression. Background is pure white (#ffffff) with occasional soft beige sections (#f8f6f3) for visual breaks. Text is near-black (#1a1a1a) rather than pure black for reduced eye strain. This palette communicates professionalism, maturity, and thoughtful restraint.

**Layout Paradigm**:
Strict vertical rhythm with generous margins. Content is organized in a single-column layout (max-width 720px) for optimal readability, with occasional full-width image breaks. Sections are separated by large vertical spacing (120-160px) creating a sense of breathing room. The hero section is minimal—name and title centered with ample space above and below. Scrolling feels like turning pages in a magazine, with clear section breaks and thoughtful pacing.

**Signature Elements**:
- Thin horizontal rules (1px, 20% opacity) as section dividers
- Circular profile image with subtle shadow
- Small, uppercase labels in accent color for section headings

**Interaction Philosophy**:
Interactions should be understated and elegant, never calling attention to themselves. Hover states produce gentle color shifts (green → terracotta) over 400ms. Links underline on hover with a smooth width transition. Buttons have no background until hovered, when they fill with color. The overall feeling is calm, confident, and refined—interactions enhance rather than distract.

**Animation**:
Extremely subtle, almost imperceptible animations. Content fades in with minimal upward motion (8px translate) over 600ms. No scroll-triggered animations except for a gentle parallax effect on the hero section (0.3 speed multiplier). Hover transitions use ease-out timing at 400ms. Page transitions are simple cross-fades. The philosophy is "animation should be felt, not seen"—creating smoothness without spectacle.

**Typography System**:
- Display: Fraunces (600, 700) for large headings—a serif with character
- Body: Inter (400, 500) for all content and UI
- Accent: Inter (600) for labels and metadata, all-caps with increased letter-spacing
- Hierarchy through size, weight, and spacing rather than font changes
- Generous line height (1.8) for body text, 1.3 for headings
- Comfortable reading size (18px base) for body text
- Ample paragraph spacing (1.5em) for editorial pacing
</text>
